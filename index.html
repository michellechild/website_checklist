HTML

  • !DOCTYPE html is included at the top of the html document.
  • A language is specified (lang="en-US")
  • A meta tag is specified (charset="utf-8")
  • Reset CSS file is before the Styles CSS file.
  • Br tags are used for type purposes only.
  • Alt tags are included for all images.
  • H1 tag is only used once.
  • Content rests inside a container div.
  • All characters are coded.
  • Nothing resides between ul tags and li tags.
  • Id tags used only once.
  • Class and id tags are labeled based on content.
  • Asset files are referenced correctly.
  • All tags are closed
  • HTML document is indented and spaced correctly.

CSS

  • Elements, classes and ids categorized correctly.
  • Category Examples:
    • Grid
    • Universal Fixes
    • Typography
    • Buttons
    • Header
    • Nav
    • Sections
    • Footer
  • A CSS reset or normalize file is used.
  • Units of measure are correctly labeled.
  • All brackets are closed.
  • CSS document is indented and spaced correctly.
  • Comments and spacing is used to separate and define styles.

Technical Specifications and File Management

  • No spaces are used in any file name, folder, or anything. ever. never.
  • First file is named index.html.
  • Stylesheets are in a separate folder.
  • Asset files are in a separate folder.
  • Asset files are labeled clearly and based on content.
  • Js files are in a separate folder.
  • All unnecessary files are removed.

Content

  • Content belongs to you OR
  • The author has been credited.
  • Content is clearly written and easy to understand.
  • Content is the focus of the site.
  • Content uses consistent styling.

Design Foundations

Grid

  • Site is (mostly) aligned to a grid.
  • There is a sufficient amount of whitespace between elements of the page.
  • Gestalt Theory Principles were considered with the overall design.
    • Law of Pragnanz
    • Figure/Ground
    • Similarity
    • Unity
    • Continuity
    • Closure
    • Proximity
    • Common Region
    • Synchrony
    • Symmetry
    • Parallelism
    • Past Experience

Typography

  • Typefaces work well together.
    • X-heights match.
    • Effectively sets the tone for the content.
  • A max-width is set for type allowing 40-90 characters per line.
  • The following five properties are styled for each typographic element in your CSS:
    • Font-family, including second, third and fourth options.
    • Font-size (at least 16px for web).
    • Line-height (140% minimum, but ultimately based on font-size.).
    • Color.
    • Max-width: 45 to 90 characters per line.

Color Theory

  • Colors reflect the message of your content.
  • The color contrast of your type and background colors has passed the test.
  • The site has been tested for color blindess compatibility.

Style Tiles

  • The site uses style tile elements consistently, repeatedly and correctly.
  • Style tiles concentrate on style, and do not look like a website.
  • Style Tiles include these elements:
    • Colors
    • Textures
    • Buttons
    • Icons
    • Arrows
    • Type Specimen
    • Divider
    • Logo

Git/Terminal

  • Work locally!
  • Site is committed to GitHub as changes are made
  • Commit messages are clear and expressive
  • GH-Pages are updated with each commit
  • Descriptive README files are included in each repository.

Sass

  • Sass --watch is always turned on when working in .scss files.
  • Sass files attempt the DRY (Don't Repeat Yourself) philosophy.
  • The Inception Rule is followed (no more than four levels deep!).

Saving Images

  • This process was followed when saving images for the web:
    1. Images are sized proportionally.
    2. Images are proportionally and carefully cropped to fit size specifications.
    3. The best filetype for your image is chosen.
    4. The Save For Web feature on Adobe Illustrator or Photoshop is used.
