HTML

  • Start your document like this: <!DOCTYPE html>
    <html lang="en-US">
    <head>
    <title>Web Design is Cool</title>
  • Link to your CSS stylesheets. <link href="stylesheets/style.css" type="text/css" rel="stylesheet">
  • Include a meta tag for charset to declare the character encoding used of the page. <meta charset="utf-8">
  • Include a meta tag for viewport to allow devices to read media queries (see responsive web design). <meta name="viewport" content="width=device-width, initial-scale=1.0">
  • Link to web fonts from services like Adobe TypeKit, myFonts, Webtype, etc. (Adobe Typekit example here). <script src="//use.typekit.net/fxv0szo.js"></script> <script>try{Typekit.load();}catch(e){}</script>
  • And finally, link to your .js file or jQuery hosting site (this can also be placed at the bottom of your body). <scriptsrc="//code.jquery.com/jquery-1.11.0.min.js"></script><head>
  • Use <br> tags for type purposes only.
  • Include <alt> tags on all images.
  • Only use <h1> tag is once per page.
  • Wrap a container div around content!
  • Code all characters: & ‘ ’ “, ”
  • Make sure nothing resides between <ul> and <li>tags.
  • Only use <id> tags once.
  • Indent and space your HTML document correctly. If you're using Sublime Text, try the package HTML Beautify.

CSS

  • Use a CSS reset or normalize file.
  • Label slass and id tags based on content, not style.
  • Indent and space stylesheet correctly.
  • Organize stylesheet into sections and label the sections with comments.

Sass

  • Label Variables and Mixins based on content, not style.
  • Begin imported files with an underscore.
  • Always turn on “Sass --watch” when working in .scss files.
  • Use nesting cautiously to avoid specificity issues—if you don't need to nest, don't.
  • Follow the The Inception Rule—no more than four levels deep!.

jQuery

  • Link to the jQuery hosting site in the head (or bottom of the body) of your HTML. <scriptsrc="//code.jquery.com/jquery-1.11.0.min.js"></script><head>
  • Place your javascript/jquery in a separate file and link to the file at the bottom of your body tag in HTML, to allow all HTML and CSS to load before your javascript/jquery.
  • If you only have a few lines of jquery/js, you can place it directly in your HTML, at the bottom of the body tag, using a <script&rt; tag.
  • Use comments whenever possible to explain (to your future self or others) what's going on in your jquery/js.

RWD

Responsive Web Design

  • First things first. Add a meta tag for viewport to your HTML to allow devices to read media queries (see responsive web design). <meta name="viewport" content="width=device-width, initial-scale=1.0">
  • Use the “mobile-first“ (ahem, global-first) approach when designing. Meaning, design for your smallest screen size without adding a media query. Why? If someone’s browser doesn’t accept media queries, they will at least be able to see this version of your site, instead of a sad, un-styled HTML page.
  • Use ems instead of pixels for your type! If you like easy math like me, you can add font-size: 62.5%;
     to your body or html styles. Then 1em = 10 pixels.
  • Try to keep anything width-related in percents, and anything height-related in pixels. Exception: max-width must be in pixels. Here's an example of the container elements (always a good idea) I use: .container{ max-width: 1000px; width:85%; margin: 0 auto; overflow: hidden;}
  • Create re-sizable images by adding the following code to your global CSS:.img{ max-width: 100%;}
  • Want more? Checkout Sam Kapila's awesome checklist for responsive tips, resources, and examples.

Tech Specs

Technical Specifications and File Management

  • Dont' use spaces in any file name, folder, or anything. ever. never.
  • Label your first html file index.html and placed it in the root folder.
  • Place stylesheets, .js files, and asset files in separate folders.
  • Organize files and label them based on content.
  • Remove all unnecessary files.

  • OR is author credited?

  • Is the content the focus of the site?

Typography

  • Refer to Butterick's Practical Typography for all things typography.
  • Consider typefaces well, starting with body type. There are several sites and articles to help you find a good typographic pair. Try these: Creative Bloq, Type Genius, (or you can always Google it).
  • Choose heading and body type with matching or similar x-heights.
  • Use type to effectively set the tone for the content.
  • Do not use more than two font families. One superfamily is also good way to go!
  • Use ems instead of pixels for your type for greater accessibility. You can add font-size: 62.5%;
     to your body or html styles. Then 1em = 10 pixels.
  • Set a max-width for type allowing 45-90 characters per line. Use this line (in your font) as a guide (52 characters):

    abcdefghijklmnopqrstuvwxyzabcdefghijklmnopqrstuvwxyz
  • Include the following five properties for each typographic element (you can also use CSS Font Stack):
  • Font-family, including second, third and fourth options.
  • Font-size: minimum 16px (or 1.6ems) for body type.
  • Line-height (140% minimum, but ultimately based on font-size.).
  • Color (see Color Theory).
  • Max-width (see above).

Adobe TypeKit

  • Adobe Typekit is just one of several web-font services, but they all will work in a similar way as explained here.
  • First, create a kit by selecting ‘Kits’, then ‘Create new kit’. Add the domain of the site where you'll be using the web fonts. For example, my GitHub kit uses the domain michellechild.github.io Note: Codepen accounts use s.codepen.io.
  • Find a font. Select ‘Use fonts’, then the ‘Web’ tab. Add to your existing kit or create a new kit here.
  • The Typekit Editor screen will appear. Select the weights and styles you'd like to use on your site, but pay attention to the Kit Size at the bottom of the screen. Exceeding 150K will affect your site’s loading time.
  • Keep the default language support unless you need to use all characters.
  • The links at the bottom-right of each section will give you the css syntax for the font and each weight and style.You can also save a font stack under CSS stack, which will be automatically be applied when using the font.
  • When you have the weights and styles chosen, select ‘ Publish’.
  • Almost finished! Now select ‘Embed Code’ on the top righthand corner. Place this code in your the head of your html.

Color
Theory

  • Use color to reflect the message of your content.
  • Use the contrast ratio to test your type color and background color contrast level.
  • Test your site for color blindess compatibility.

Wireframes

  • Set up twelve column grid in the background of your wireframe. Even if you don't use all 12 columns, this will serve as your 2, 3, 4, or 6 column grid as well. You can download a template, or create your own.
  • Always wireframe in grayscale. This is about layout, not color.
  • If you don't like wireframing in Photoshop, checkout Omnigraffle,Balsamiq, or choose your own from Creative Bloq’s top 20 wireframing tools

Saving
Images

  • Size images proportionally.
  • Carefully proportion and crop images to fit size specifications.
  • Consider the different file types and choose the best one: .png, .jpg, .gif, or .svg.
  • Use ‘Save For Web’ on Adobe Illustrator or Photoshop.

Accessibility

  • Site loads quickly.
  • Site is responsive, but includes a global version for browsers that do not support media queries.
  • Site includes a viewport tag for mobile devices (see Responsive Web Design.
  • Use ems instead of pixels for your type for greater accessibility. You can add font-size: 62.5%;
     to your body or html styles. Then 1em = 10 pixels.
  • Consult the A11y project usability checklist.
