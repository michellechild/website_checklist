HTML

  • Your document begins like this: Web Design is Cool (1) (2) (3) (4) (5)
  • (1) Links to your CSS stylesheets.
  • (2) Meta tag for charset declares the character encoding used of the page.
  • (3) Meta tag for viewport allows devices to read media queries (responsive web design).
  • (4) Links to web fonts (if used) from services like Adobe TypeKit, myFonts, Webtype, etc.
  • (5) Links to your Js stylesheets or jQuery hosting site. This can also be placed at the bottom of your body.

  •  tags are used for type purposes only.
  • tags are included on all images.
  • h1 tag is only used once per html page.
  • Container div is set.
  • Characters are coded: &, ‘, ’, “, ”
  • Nothing resides between
      and
    • tags.
    • Id tags are used only once.
    • HTML document is indented and spaced correctly.

CSS

  • A CSS reset or normalize file is used.
  • Class and id tags are labeled based on content, not style.
  • CSS document is indented and spaced correctly.
  • Stylesheet is organized into sections and clearly labeled with comments.

Sass

  • Variables and Mixins are labeled based on content, not style.
  • Imported files begin with an underscore.
  • Sass --watch is always turned on when working in .scss files.
  • Nesting is used cautiously to avoid specificity issues— if you don't need to nest, don't.
  • The Inception Rule is followed (no more than four levels deep!).

Git/Terminal

  • Work locally!
  • Push all work to a branch until a live version can be pushed to the master.
  • Site is pushed to GitHub as changes are made.
  • Commit messages are clear and expressive.
  • Gh-pages are updated with each commit.
  • Descriptive README files are included in each repository.

Technical Specifications and File Management

  • No spaces are used in any file name, folder, or anything. ever. never.
  • First html file is labeled index.html and in the root folder.
  • Stylesheets, js and asset files are in separate folders.
  • Asset files are organized and labeled clearly and based on content.
  • All unnecessary files are removed.

Content

  • Content belongs to you OR
  • The author has been credited.
  • Content is clearly written and easy to understand.
  • Content is the focus of the site.

Grid

  • Site is (mostly) aligned to a grid.
  • There is a sufficient amount of whitespace between elements of the page.
  • Gestalt Theory Principles were considered with the overall design.

Typography

  • Refer to Butterick's Practical Typography for all things type. Seriously.
  • Typefaces are well considered, starting with body type. There are several sites and articles to help you find a good typographic pair. Try these: Creative Bloq, Type Genius, or you canGoogle it
    • X-heights match for heading and body type.
    • Effectively sets the tone for the content.
  • No more than two font families are used. One superfamily is also good way to go!
  • A max-width is set for type allowing 45-90 characters per line. Use this line to measure: abcdefghijklmnopqrstuvwxyzabcdefghijklmnopqrstuvwxyzabcd
  • The following five properties are styled for each typographic element (you can also use CSS Font Stack)):
    • Font-family, including second, third and fourth options.
    • Font-size (minimum 16px for body type).
    • Line-height (140% minimum, but ultimately based on font-size.).
    • Color (see Color Theory).
    • Max-width (see above).

Color Theory

  • Colors reflect the message of your content.
  • The color contrast of your type and background colors has passed the test.
  • The site has been tested for color blindess compatibility.

    • Saving Images

    • This process was followed when saving images for the web:
      1. Images are sized proportionally.
      2. Images are proportionally and carefully cropped to fit size specifications.
      3. The best filetype for your image is chosen: .png, .jpg, .gif, or .svg.
      4. The Save For Web feature on Adobe Illustrator or Photoshop is used.
